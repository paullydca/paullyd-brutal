WHY WEB DESIGN IS TOO SOFT

2026-03-19 CSS DESIGN

Take a look at any modern SaaS landing page today. What do you see? Soft, pill-shaped buttons taking 300 milliseconds to gently morph when you hover over them. Drop shadows rendered so softly they look like a faint fog. Gray text on a white background that requires squinting just to read the copyright.

Since when did web interfaces become pillows?

We've lost our way. The internet used to be structural. When you clicked a button in Windows 95, you damn well knew you clicked it. The borders inverted instantly. The contrast was harsh, legible, and functional. It was software, not an ambient art installation.

THE RETURN TO CONCRETE

That's why I've adopted Brutalism for my own work. It's an anti-design philosophy drawn from 1950s architecture.

CORE RULE: NO BLUR

If it casts a shadow, that shadow is solid #111827 offset by exactly 4 pixels. Do not attempt to use rgba(0,0,0,0.1). Real objects cast hard shadows.

When everything transitions instantly (0ms), the UI feels infinitely faster. Your users aren't waiting for a buttery CSS animation to finish before the browser registers their cognitive input.

IMPLEMENTATION NOTES

Setting up Brutalism is easy. Rip out your CSS frameworks. Define a harsh color palette (pure white, deep black, stark red). Remove all border-radius variables. Force outline states to a solid 3px offset. Do this, and you strip away the fat.

Don't design for screenshots. Design to be used. Let's make the web raw again.
